# v9模型全面升级 (2026-06-19)

## 用户核心反馈
1. **验证场次太少** — 27场完赛仅验证~10场
2. **球员数据缺失** — 孙兴慜/李刚仁数据矛盾，缺少俱乐部成就/来美适应/比赛上下文
3. **裁判维度不足** — 缺少肢体对抗容忍度和红牌崩盘链路
4. **地理优势未量化** — 梅西提前来美/南美球队群体优势/亚洲美洲困境
5. **球员上下文缺失** — 孙兴慜MLS 0球=拿不住球, 李刚仁出场少=状态保养好+欧冠心态

## 完成的改进

### 1. 全量回测 (MODEL_VALIDATION_BACKTEST.md)
- 27场逐一分析，分类为强队正路(44%)/平局(37%)/冷门(11%)/屠杀(15%)
- 模型方向正确率: 12/17=71%(有记录), 精确比分命中率: 1/17=6%
- 发现: v8方向正确率仅33%(1/3), 过度修正导致

### 2. 球员数据库v7 (player_database_v7.json)
- **修复孙兴慜数据矛盾**: 统一为LAFC 2026赛季13场0球8助(29射0中)→进球荒但角色转型为组织者
- **修复李刚仁数据矛盾**: 统一为PSG 27场(18首发)1519分钟3球4助, UCL 0首发→欧冠冠军心态+体力充沛
- **新增字段**: club_achievements, arrival_advantage, playing_context, physical_style

### 3. 裁判数据库v7 (referee_database_v7.json)
- **新增physical_tolerance**(1-10分): Tejera=2(极低), Al-Jassim=2(极低), Taylor=7(较高)
- **新增collapse_risk**: 极高(Tejera每3场1红)/高/中/低
- **崩盘链路**: 严苛裁判+防守弱队→上半场黄牌→下半场疲劳→第二黄/红→少人→连续丢球

### 4. 地理优势模块 (location_advantage_v1.json)
- 球队地理评分: 墨西哥+0.85, 美国+0.80, 阿根廷+0.75, 加拿大+0.65, 巴西+0.65
- 球员个体美洲适应加成: MLS球员+0.08, 南美在美洲+0.05, 欧洲首次来美-0.02
- 亚洲球队默认-0.10(日本例外), 非洲球队默认+0.05

### 5. MODEL_RULES.md更新到v9
- 权重调整: 地理12%→14%, 趋势8%→12%, 赔率7%→4%, 新增历史战意5%
- R1-R17规则整合

- 冷门检测: 29% → 目标50% (+21%)
